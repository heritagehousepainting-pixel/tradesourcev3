# VISUAL_REFERENCES.md

## Purpose

This file translates visual inspiration into usable implementation direction for TradeSource.

TradeSource is NOT the same product as the inspiration references below.
These references are for:
- composition
- spacing
- hierarchy
- layout confidence
- visual polish
- trust framing

Not for:
- business model copying
- wording copying
- product logic copying
- identity copying

---

## Primary Inspiration

### Opendoor
Use Opendoor as inspiration for:
- confident section spacing
- clean hero composition
- simple, premium layout rhythm
- strong content grouping
- polished marketing-page structure

Do NOT copy:
- real estate framing
- consumer-homeowner tone
- house/offer language
- business logic

---

## What To Borrow From References

### From Premium B2B Product Pages
Borrow:
- stronger nav structure
- cleaner CTA hierarchy
- tighter type scale
- more intentional whitespace
- better card composition
- cleaner footer architecture

### From Premium Dashboards
Borrow:
- information grouping
- action clarity
- strong status treatment
- cleaner scan path
- higher signal density without clutter

---

## Nav Reference Intent

The nav should feel:
- stable
- premium
- aligned
- intentionally spaced
- like a product company, not a prototype

Use references that show:
- strong left/right balance
- obvious primary CTA
- low-clutter nav links
- consistent height and padding
- dark or neutral premium styling

---

## Hero Reference Intent

The hero should feel:
- contractor-first
- serious
- premium
- B2B
- operational
- confident

Use references that show:
- strong headline/subheadline balance
- text block that is not too wide
- CTA row with clear priority
- trust/support row beneath CTA
- clean visual block that supports rather than overwhelms

Avoid:
- oversized empty visuals
- stock-illustration feel
- abstract gimmicks
- giant decorative shapes without value

---

## Dashboard Reference Intent

The dashboard should feel:
- like an operating center
- calm and high-signal
- grouped by job state and action
- visually clean but dense enough to be useful

Use references that show:
- clean panel hierarchy
- status-first layout
- strong action surfaces
- clean label systems
- restrained but premium visual treatment

Avoid:
- generic admin templates
- cluttered SaaS boxes everywhere
- weak typography
- equal-weight panels with no clear focus

---

## Footer Reference Intent

The footer should feel:
- structured
- calm
- polished
- intentional

Use references that show:
- clean column layout
- restrained hierarchy
- consistent link treatment
- dark-theme compatibility

Avoid:
- raw link dumps
- underdesigned legal clutter
- accidental white-background footer feel

---

## Screenshot Use Rule

When screenshot references are available:
- implement the feel and structure
- do not clone the page
- translate the layout logic into TradeSource’s own product language

When screenshots are not available:
- default to the intent rules in this file
- keep composition restrained and serious
- favor clarity over experimentation

---

## Notes For Section-by-Section Work

If working on one section:
1. identify the section type
2. use the matching intent above
3. implement the structure and hierarchy first
4. then refine spacing, type, and CTA treatment
5. do not redesign unrelated sections in the same pass unless necessary
