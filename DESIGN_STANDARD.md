# DESIGN_STANDARD.md

## Purpose

This file defines the non-negotiable design standard for all TradeSource product work.

TradeSource must never feel like a toy app, a generic AI-generated dashboard, or a rough MVP with placeholder styling. It must feel serious, trusted, modern, premium, and operationally credible from the first second.

The standard is not “good enough to function.”
The standard is “good enough to earn trust quickly from real contractors, real business owners, and real users under real conditions.”

---

## Core Design Principle

TradeSource must look and feel like a real company with real standards.

Every screen should communicate:
- trust
- clarity
- professionalism
- control
- seriousness
- product maturity

Design must reduce doubt.
Design must reduce friction.
Design must increase confidence.

If a screen feels:
- sloppy
- generic
- over-decorated
- confusing
- thin
- toy-like
- unfinished
- “AI made this in five minutes”

then it fails this standard.

---

## Brand Feel

TradeSource is not playful.
TradeSource is not startup-cute.
TradeSource is not social-media-loud.
TradeSource is not built on gimmicks.

TradeSource should feel:
- private
- professional
- verified
- operational
- trustworthy
- high-signal
- clean
- quietly premium

Visual tone should lean toward:
- B2B trust
- infrastructure
- control panel quality
- premium contractor network
- serious marketplace
- enterprise-grade confidence without feeling corporate-dead

---

## First Impression Standard

A user must understand within seconds:
1. what this is
2. who it is for
3. why it matters
4. why it can be trusted
5. what action to take next

The first screen must not be vague.
The first screen must not hide the value.
The first screen must not make the user think too hard.

The first impression must feel:
- clear
- strong
- intentional
- visually polished
- credible enough to continue

If the top of the page does not create immediate confidence, it is not done.

---

## Trust Standard

Trust is the highest design priority.

The product must visually communicate:
- vetting
- seriousness
- order
- quality control
- clear structure
- consistent standards
- low chaos

Trust is damaged by:
- fake-looking stats
- clutter
- inconsistent spacing
- weak typography
- poor hierarchy
- random colors
- overcrowded cards
- unclear buttons
- awkward responsiveness
- broken states
- empty visual shells
- generic icons
- overuse of gradients or flashy effects

Every screen must make the user feel:
“This is controlled.”
“This is real.”
“This is safe to use.”
“This feels built for professionals.”

---

## Visual Hierarchy Rules

Each screen must have a clear hierarchy.

The user should instantly know:
- primary message
- primary action
- secondary action
- supporting information
- lower-priority details

Hierarchy must be created through:
- spacing
- typography
- alignment
- grouping
- scale
- contrast
- restraint

Do not rely on decoration to create hierarchy.
Do not make everything equally loud.
Do not create visual competition between elements that should not compete.

One screen.
One clear priority.
One clear next step.

---

## Layout Standard

Layouts must feel structured, stable, and intentional.

Requirements:
- strong alignment
- consistent spacing rhythm
- predictable section grouping
- no floating randomness
- no crowded blocks
- no accidental imbalance
- no dead zones created by weak composition

All pages must feel designed, not merely assembled.

Use whitespace deliberately.
Do not fear breathing room.
Do not cram content to prove density.
Dense is not the same as powerful.

The layout should feel calm, sharp, and controlled.

---

## Typography Standard

Typography must carry authority.

Type must be:
- legible
- restrained
- consistent
- modern
- clean
- easy to scan

Avoid:
- weak hierarchy
- tiny text
- oversized body copy
- too many font weights
- decorative styling
- low-contrast text
- poor line spacing
- crowded labels
- inconsistent title treatment

Headings should create structure.
Body copy should create clarity.
Labels should support speed.

Typography should help the product feel expensive, not busy.

---

## Color Standard

Color must be controlled and purposeful.

Use color to support:
- hierarchy
- status
- trust
- action clarity
- state communication

Do not use color to entertain.
Do not use color just to make things “pop.”
Do not use too many accents.
Do not make the interface loud.

The palette should feel:
- calm
- refined
- professional
- stable
- deliberate

Status colors must be used consistently.
Interactive colors must be clear.
Background and surface colors must preserve readability and trust.

If color use feels trendy, flashy, or noisy, reduce it.

---

## Component Standard

All components must feel systemized.

Buttons, cards, inputs, badges, tabs, tables, modals, and navigation must all feel like they belong to the same product system.

Component quality requirements:
- consistent spacing
- consistent radius strategy
- consistent border logic
- consistent hover and active behavior
- clear focus states
- clear disabled states
- clear error states
- clear loading states
- clean responsive collapse behavior

No component should feel custom in a bad way.
No component should feel copied from a random starter template without adaptation.

Every component should feel intentional and production-ready.

---

## Interaction Standard

Interactions must feel clean and reliable.

Requirements:
- obvious click targets
- no ambiguous actions
- no fragile UI behavior
- smooth transitions where useful
- no unnecessary movement
- no distracting animation
- loading states that explain what is happening
- error states that help the user recover
- success states that confirm progress clearly

Interaction design should increase confidence, not spectacle.

The product should feel:
- responsive
- stable
- dependable
- under control

---

## Responsive Design Standard

TradeSource must work credibly across:
- desktop
- laptop
- tablet
- mobile

Responsive design is not just about fitting.
It is about preserving quality.

Each viewport must maintain:
- hierarchy
- readability
- alignment
- spacing
- trust
- usability

No layout should collapse into chaos.
No critical action should become hard to find.
No important section should become visually broken, cramped, or obviously compromised.

Mobile should feel intentionally designed, not like a compressed desktop leftover.

---

## Dashboard Standard

Dashboards must feel operational, not decorative.

A dashboard should communicate:
- status
- clarity
- momentum
- control
- decision readiness

It must not feel like:
- a template
- a fake analytics board
- empty admin filler
- random stat cards with no meaning

Every metric, status, card, or section must have a reason to exist.

If data is shown, it must feel useful.
If a card is present, it must support user understanding or action.
If a section does not matter, remove it.

---

## Landing / Marketing Screen Standard

The landing experience must immediately answer:
- what is TradeSource
- who is it for
- why it matters
- why it is credible
- what to do next

It must not rely on vague startup language.
It must not hide behind generic headlines.
It must not feel overhyped.

The page should be strong, clear, and premium.
It should create trust before persuasion.

The design should make the user feel:
“This is legitimate.”
“This solves a real problem.”
“This looks like a serious network.”

---

## Marketplace / Workflow Screen Standard

Workflow screens must optimize for action clarity.

On any screen involving:
- job posting
- applications
- vetting
- messaging
- reviews
- status transitions
- dashboards
- approvals

the user should instantly understand:
- where they are
- what they can do
- what happens next
- what state the job or process is in

Workflow confusion is unacceptable.

Do not sacrifice clarity for visual novelty.

---

## Iconography Rule

Use clean, professional iconography only.

Do not use:
- emoji as core UI
- playful symbol language
- generic Apple-style shortcut aesthetics
- random icon mixtures
- decorative icons with no functional purpose

Important rule:
NO APPLE EMOJIS OR GENERIC APPLE ICONS.

Icons must support the interface, not dominate it.

---

## Copy + Design Relationship

Design and copy must reinforce each other.

The UI should not try to save weak copy.
The copy should not try to save weak UI.

Every key screen should have:
- crisp headlines
- clear labels
- strong calls to action
- credible supporting text
- no fluff
- no filler
- no fake urgency
- no awkward AI phrasing

The interface must feel written by adults for adults.

---

## “No Toy App” Rule

TradeSource fails this standard if it looks like:
- a quick prototype
- a student project
- a startup template with renamed labels
- a shallow AI-generated SaaS dashboard
- a product that has not been pressure-tested
- a pretty shell without authority

It must look like something a real contractor or operator could trust with actual work.

---

## “Show Real Seriousness” Rule

Every major screen should survive this question:

“If something went wrong on a real job, would this product still feel trustworthy enough to rely on?”

If the answer is no, the design is not ready.

Design must make the product feel:
- accountable
- real
- stable
- worth returning to
- worth trusting under pressure

---

## Required Design Review Checklist

Before any screen is considered done, check:

- Is the purpose obvious within seconds?
- Is the hierarchy clear?
- Does the screen feel trustworthy?
- Does it avoid generic AI-app energy?
- Does it feel premium without trying too hard?
- Is the spacing clean and consistent?
- Is the typography sharp and readable?
- Do the colors feel controlled and professional?
- Are the components systemized?
- Is the primary action obvious?
- Is the mobile version still high quality?
- Would a real user think this is serious?
- Would the founder be proud to show this publicly?
- Does this increase confidence in the product?
- Does this screen deserve to exist as-is?

If any of those answers are weak, revise.

---

## Design Failure Conditions

A screen fails automatically if:
- CSS is broken
- layout collapses
- mobile feels compromised
- hierarchy is confusing
- trust signals are weak
- visuals feel cheap
- fake data undermines credibility
- the page feels unfinished
- the design looks generic or template-heavy
- the screen creates doubt instead of confidence

---

## Final Standard

Do not aim for “acceptable.”
Do not aim for “pretty enough.”
Do not aim for “MVP quality.”

Aim for:
professional, clear, trustworthy, premium, and real.

TradeSource must feel like a serious product in a serious market for serious users.
Anything less is not finished.
